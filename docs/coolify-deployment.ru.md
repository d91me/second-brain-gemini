# Деплой в Coolify v4

Данное руководство описывает процесс развертывания проекта **second-brain-gemini** на сервере с Coolify v4 (включая v4.0.0-beta.462 и новее).

Проект поставляется с готовым `Dockerfile`, поэтому использование Coolify максимально упрощено и не требует `docker-compose.yml`.

## Шаг 1: Создание ресурса в Coolify

1. Откройте панель управления Coolify.
2. Перейдите в **Projects** (Проекты) -> выберите/создайте проект и окружение (например, `Production`).
3. Нажмите **+ New Resource** (Добавить ресурс) -> выберите **Git Repository** (если ваш код на GitHub/GitLab).
4. Выберите нужный репозиторий `second-brain-gemini` и укажите ветку (например, `main`).
5. В качестве Build Pack (типа сборки) выберите **Dockerfile**.

## Шаг 2: Настройка переменных окружения (Environment Variables)

После создания ресурса, не спешите нажимать кнопку Deploy. Перейдите во вкладку **Environment Variables** (Переменные окружения) и добавьте следующие ключи:

| Ключ | Значение (Пример) | Описание | Обязательно |
|------|-------------------|----------|-------------|
| `TELEGRAM_BOT_TOKEN` | `123456789:ABCdefGHI...` | Токен вашего телеграм-бота от @BotFather | Да |
| `GEMINI_API_KEY` | `AIzaSy...` | API ключ от Google Gemini | Да |
| `DEEPGRAM_API_KEY` | `abc123def456...` | API ключ от Deepgram (для распознавания голоса) | Да (если нужны голосовые) |
| `TODOIST_API_KEY` | `e2b5...` | API ключ от Todoist (для MCP интеграции через Gemini CLI) | Да (если нужен Todoist) |
| `ALLOWED_USER_IDS` | `[123456789]` | JSON-массив с ID пользователей Telegram, которым разрешен доступ к боту | Да |
| `VAULT_PATH` | `./vault` | Путь к папке Vault (оставьте значение по умолчанию `./vault`) | Да |

> [!TIP]
> Убедитесь, что галочка "Build Variable" выключена для этих секретов, чтобы они не попали в кэш сборки Docker, а использовались только во время работы контейнера.

## Шаг 3: Настройка хранилища (Storages / Volumes)

Чтобы ваши данные конфигурации и база знаний (Vault) не удалялись при перезапуске контейнера, настройте постоянные хранилища во вкладке **Storages**:

1. **Vault (база знаний)**
   - Name/Source: `second-brain-vault` (или любой другой)
   - Destination: `/app/vault`
   
2. **Gemini CLI Settings (кэш и настройки Todoist)**
   - Name/Source: `second-brain-gemini-settings`
   - Destination: `/root/.gemini`

*Примечание: Если настройки Todoist задаются локально в `.gemini/settings.json`, они запекаются в образ через Dockerfile на этапе сборки. Однако подключение Volume позволит вам менять эти файлы на сервере без пересборки (через вкладку Storages).*

## Шаг 4: Деплой

1. Вернитесь во вкладку **Configuration**.
2. В поле `Start Command` (если есть) можно оставить пустым, так как Dockerfile уже содержит команду `CMD ["python", "-m", "d_brain"]`.
3. Нажмите **Deploy**.

Coolify начнет клонирование репозитория, сборку образа на основе `Dockerfile` и запуск контейнера. Процесс сборки можно отслеживать во вкладке **Deployments**.

## Проверка работы

После успешного деплоя (статус `Healthy` / `Running`):
- Перейдите к вашему боту в Telegram и отправьте команду `/start`.
- Если бот отвечает, значит деплой прошел успешно!
- Логи работы бота можно посмотреть прямо в Coolify во вкладке **Logs**.
